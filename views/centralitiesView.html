Degree

Degree centrality measures the importance of a node by the degree of the node, i.e. the number of edges incident to the node (which is euivalent to the number of its neighbours). The higher degree of a node, the more important the node is. Formally, for all $v \in V$: \[c_{\mathit{degree}}(v) = |(v)|,\] where $N(v)$ denotes the set of neighbours of $v$.
Wikipedia: degree (graph theory)

Closeness

Closeness centrality quantifies importance of a node in terms of shortest distances from this node to all other nodes in the graph. The most important node is the one with the shortest average path length to all other nodes. Formally, for all $v \in V$: \[c_{\mathit{closeness}}(v) = \frac{1}{\sum{w \in V} d(w,v)},\] where $d(w,v)$ denotes the length of a shortest path from $w$ to $v$.
Wikipedia: centrality [ https://en.wikipedia.org/wiki/Centrality ]

Betweenness

Betweenness centrality quantifies the importance of a given node in the context of network flow. If we consider all the shortest paths in the network, then the greater the proportion of such paths that goes through a given node, the more important the role of that node in the network. Formally, for all $v \in V$: \[c_{\mathit{betweenness}}(v) = \sum_{s \neq v \neq t \in V} \frac{|\{sp(s,t): v \in sp(s,t)\}|}{|\{sp(s,t)\}|},\] where $sp(s,t)$ denotes the shortest path between $s$ and $t$.

Eigevector Centrality

Eigenvector centrality ranks nodes based on the concept that connections to higher-ranked nodes contribute more to the ranking of the node in question than equal connections to lower-ranked nodes. To presents its formal definition, let us denote by $A$ the adjacency matrix of network $G=(V,E)$, \ie~$A_{i,j}=1$ when $(i,j) \in E$. Eigenvalue of matrix $A$ is $\lambda \in \mathbb{R}$ such that there exists corresponding eigenvector $x \in \mathbb{R}^{|V|}$ for which $A x = \lambda x$. Normalized eigenvector centrality of a node $v$ in a graph $G$ can be expressed as: \[ c_{\mathit{eigenvector}}(v) = \frac{x_v}{\sum_{w \in V} x_w} \] where $x$ is the eigenvector corresponding to the largest eigenvalue of the adjacency matrix of $G$.

Game-theoretic Centrality Measures


Game-theoretic measures use the combinatorial structure of coalitional games to get more refined understanding of which nodes are the most important in the network. To define the game-theoretic centrality we first need to specify the characteristic function, i.e., the value of any group (subset) of nodes that corresponds to centrality of this group. Next, we use a game-theoretic solution concept of choice (e.g., the Shapley value or the Banzhaf Index) to get a centrality scores of individual nodes.

The key advantage of this approach is that nodes are ranked not only according to their individual roles in the network but also according to how they contribute to the role played by all possible subsets of nodes.

For more details and applications see our website www.game-theoretic-centrality.com.

SV-based degree centrality

The value of a coalition $C \subseteq V$ is defined as the number of neighbours of $C$. Note that $C$ does not have to necessarily connected. Formally: \[ \nu(C,G) = \begin{cases} 0 & \mbox{ if } C = \emptyset \\ |N(C)| & \mbox{ otherwise.} \end{cases} \] where $N(C) = \{v \in V\setminus C: \exists w \in C \ (v,w) \in E\}$.
For more see Michalak et al. (2013) , Algorithm 1, page 622.

The value of a coalition $C \subseteq V$ is defined as the number of neighbours of $C$ that are not further than $k$ steps away. Formally:\[ \nu(C,G) = \begin{cases} 0 & \mbox{ if } C = \emptyset \\ |N^k(C)| & \mbox{ otherwise.} \end{cases} \] where $N^k(C) = \{v \in V\setminus C: \exists w \in C, \ sp(v,w)\leq k\}$, i.e., it is the set of all such nodes from $V \setminus C$ that can be reached from at least one node in $C$ with at most $k$ steps.

Note that for $k=1$, the extended SV-degree centrality collapses to the SV-degree centrality.
For more see Michalak et al. (2013) , Algorithm 2, page 624.

SV-degree (general, weighted networks)

The value of a coalition $C \subseteq V$ is defined as the sum of the shotest distances between $C$ and all other nodes in the network. Formally: \[ \nu(C,G) = \begin{cases} 0 & \mbox{ if } C = \emptyset \\ \sum_{v \in V\setminus C}f(min_{w \in C}\{sp(w,v)\}) & \mbox{ otherwise,} \end{cases} \] where $min_{w \in C}\{sp(w,v)\}$ is the minimum distance of node $v \in V \setminus C$ from $C$ and $f$ is an arbitrary function.

SV-based betweenness

The value of a coalition $C \subseteq V$ is defined as the fraction of shortest paths passing through $C$. Formally: \[ \nu(C,G) = \sum_{ s,t \notin C } \frac{|\{sp(s,t): C \cap sp(s,t) \neq \emptyset \}| }{|\{sp(s,t)\}|} \] where $sp(s,t)$ denotes the shortest path between $s$ and $t$.
For more see Szczepański et al. (2012) , Algorithm 1.

Myerson

Myerson (1977) considered a coalitional game restricted by a graph each edge of which represents the communication link between two agents. In the Myerson model, only if players are connected, either directly via an edge or indirectly via intermediaries, can they effectively cooperate in a coalitional game and generate the value added.

Mathematically, for the value function $v$ that specifies values of all connected coalitions, the Myerson value of node $v_i$ is given by: \[ MV_i(\nu,G)= SV_i(\nu^G) \]
